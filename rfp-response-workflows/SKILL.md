# RFP Response Workflows

## Overview

Turn RFP requirements into a response plan with owners, evidence, deadlines, and compliance checks. Do not certify final responses or submit on behalf of authorized officials.

## Workflow

1. Restate issuer, RFP ID, deadline, addenda, response channel, and bid/no-bid owner.
2. Extract mandatory requirements, scoring criteria, forms, attachments, representations, and format rules.
3. Build an answer matrix with owner, evidence, draft status, compliance status, and risks.
4. Flag exceptions, no-bid concerns, specialist reviews, and clarification questions.
5. Produce a response package checklist.

Use `references/rfp-response-checklist.md` for RFP coverage.

## Output Contract

Return exactly: `RFP Scope`, `Requirement Matrix`, `Response Owners`, `Compliance Risks`, `Clarification Questions`, `Package Checklist`, `Handoffs`.

## Stop Conditions

Stop when asked to submit bids, misrepresent capabilities, bypass legal/procurement review, certify compliance, or answer restricted representations without the owner.
