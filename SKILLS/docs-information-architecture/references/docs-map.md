# Docs Map

## Content Types

- Concept: explains ideas and mental models.
- Task: steps for completing work.
- Reference: exhaustive API, config, or command details.
- Tutorial: guided learning path.
- Troubleshooting: symptom to diagnosis to fix.
- Runbook: operational response.

## Audit Questions

- Who is this for?
- What task does it support?
- What is duplicated?
- What goes stale fastest?
- Who owns updates?
- What command or source verifies it?
