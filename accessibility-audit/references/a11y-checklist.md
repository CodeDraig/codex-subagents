# Accessibility Checklist

## Core Checks

- Keyboard can reach and operate everything.
- Focus order matches visual and task order.
- Focus is visible and not trapped.
- Controls have accessible names.
- Semantic elements are used before ARIA.
- Dynamic updates are announced when necessary.
- Errors identify fields and recovery.
- Color contrast is sufficient.
- Motion can be reduced.
- Touch targets are large enough.

## Severity

- Critical: cannot complete task.
- High: major barrier or misleading state.
- Medium: workaround exists but costly.
- Low: improvement or edge case.
